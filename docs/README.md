# Project Documentation

This folder contains current project notes only. Historical pass-by-pass audit files were removed after the FastAPI/React refactor was aligned with `main`.

## Current Stack

- FastAPI backend in `backend/`
- React + Vite + TypeScript frontend in `frontend/`
- PostgreSQL as the only supported application database
- Local PostgreSQL dev stack in `infra/dev/`
- Legacy PHP/HTML/CSS/JS kept only under `to_delete/` as a temporary reference

## Canonical Database Shape

Runtime code now targets the normalized PostgreSQL schema from `main`:

- tables: `map`, `vehicle`, `player`, `tuning_part`, `tuning_setup`, `tuning_setup_part`, `world_record`, `pending_submission`, `news`
- column names: lowercase `snake_case`
- IDs generated by PostgreSQL sequences and `RETURNING`
- `news` and `pending_submission` are canonical; legacy `_news` and `_pendingsubmission` are migration inputs only

Public API responses still expose the camelCase keys expected by the frontend and PHP-compatible routes, such as `idMap`, `nameVehicle`, and `idTuningSetup`.

## Supported Application Workflows

- public data browsing for maps, vehicles, players, tuning parts, tuning setups and records
- records filters, sorting, sharing and CSV export
- news display
- public record submission with hCaptcha validation
- Discord-cookie based admin access
- admin record submit, delete, questionable status and tuning setup assignment
- admin catalog additions, including optional SVG icon upload
- pending submission approval and rejection
- news posting
- maintenance mode controls
- database integrity check
- PostgreSQL application backup create, list, download and delete

## Not In The Browser Admin

PostgreSQL restore and arbitrary SQL import are intentionally not implemented as browser admin actions. They are destructive operational tasks and should be handled with reviewed database tooling such as `pg_dump`, `pg_restore`, managed backups or deployment jobs.

## Verification Baseline

The current baseline was validated with:

```powershell
cd backend
.\.venv\Scripts\python.exe -m ruff check app tests
.\.venv\Scripts\python.exe -m pytest

cd ..\frontend
npm run build

cd ..
.\scripts\dev\reset-dev-database.ps1
.\scripts\dev\start-app-stack.ps1 -RestartFastApi -RestartFrontend
node .\scripts\dev\test_system_smoke.mjs
```

The smoke test covers public API routes, frontend proxying, auth status, public submit validation, public pages, records filters, news modal, dark mode, admin workflows, maintenance controls, integrity check and backups.
